## 工序相关接口

### 接口1：多条件分页查询工序

- 请求方法：GET
- url：`/process/page`
- 参数：

| 参数名      | 说明                                 |
| ----------- | ------------------------------------ |
| processCode | 工序编码，字符串，可以为空，精确查询 |
| processName | 工序名称，字符串，可以为空，模糊查询 |
| isActive    | 工序状态，1 启用，0禁用， 空表示全部 |
| page        | 页码，默认1，                        |
| size        | 每页大小，默认10                     |

- 响应结果示例

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "total": 15,
    "list": [
      {
        "id": 15,
        "processCode": "PROC-015",
        "processName": "钻孔",
        "processDescription": "在工件上钻孔",
        "remarks": "孔径公差±0.1mm",
        "isActive": 0,
        "createTime": "2023-01-10T10:10:00",
        "creatorId": 1001,  
        "updateTime": "2023-08-30T09:40:00",
        "modifierId": 1003,
        "isDeleted": 0
      },
      ........
    ],
    
  }
}
```

注意：这里的状态是isActive。

creatorId和modifierId可以自己定义一个固定值，或者默认为null

### 接口2: 新增工序

- 请求方法：POST
- url： `/process/add`
- 参数：

参数以JSON格式放在请求体中提交，参数示例：

```json
{
    "processCode":"PROCESS_002",
    "processName":"焊接",
    "processDescription":"将零部件焊接成完整组件",
    "remarks":"使用氩弧焊工艺",
    "isActive":1
}
```

- 响应结果

成功示例

```json
{
  "code": 200,
  "msg": "操作成功",
  "data":{
    "processCode":"PROCESS_002",
    "processName":"焊接",
    "processDescription":"将零部件焊接成完整组件",
    "remarks":"使用氩弧焊工艺",
    "isActive":1
      ...这里是添加成功之后的完整的工序对象
	}
}
```

失败示例：

```json
{
  "code": 500,
  "msg": "操作失败",
  "data":null
}
```

### 接口3：根据工序编号查询当个工序对象

- 请求方法：GET
- url： `/process/code/{code}`
- 参数： 参数是放在请求路径中的 `{code}`要查询的工序的编号
- 响应示例：

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "id": 8,
    "processCode": "PROC-008",
    "processName": "包装",
    "processDescription": "成品包装准备发货",
    "remarks": "防震防潮处理",
    "isActive": 1,
    "createTime": "2023-01-10T09:35:00",
    "creatorId": 1001,
    "updateTime": null,
    "modifierId": null,
    "isDeleted": 0
  }
}
```
