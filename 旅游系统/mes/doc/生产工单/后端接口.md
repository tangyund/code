### 接口1：多条件分页查询工单列表

- 请求方法：GET
- url: `/productionOrder/list`
- 请求参数：

| 参数                  | 说明                              |
| --------------------- | --------------------------------- |
| pageNum               | 查询页面，默认为1                 |
| pageSize              | 每页大小，默认为10                |
| productionOrderCode   | 工单编码，可以为空,精确搜索       |
| productionOrderName   | 工单名称，模糊搜索                |
| productionOrderSource | 来源单据，（订单编号），可以为空  |
| materialCode          | 物料/产品编号,可以为空            |
| materialName          | 物料/产品名称，可以为空，模糊查询 |
| customerCode          | 客户编号，可以为空                |
| productionOrderType   | 工单类型，整形，可以为空          |
| requestDate           | 需求日期，可以为空                |

- 响应结果示例

```json
{
    "code":200,
    "msg":"ok",
    "data":{
        "total":158,
        "list":[
            {

                "productionOrderQuantity":1500,

}
```

### 接口3：新增功能单

- 请求方法: POST
- url: `/productionOrder/add`
- 参数：参数在请求体中，以JSON格式发送

参数示例:

```json
		{
                "productionOrderCode":"MO202503120008",
                "productionOrderName":"博世螺丝刀【一字型，蓝色刀柄】",
                "productionOrderSource":1,
                "productionOrderType":2，
                "orderCode":"202503181001",
                "materialCode":"IF2022082439",
                "productionOrderQuantity":1500,
                "customerCode":"C00197",
                "requestDate":"2025-09-08",
                "qrCodeUrl":""
            }
```

- 响应结果

正确

```json
{
    "code":200,
    "msg":"新增工单成功"
}
```

失败1：

```json
{
    "code":500,
    "msg":"工单编号重复"
}
```

失败2：

```json
{
    "code":500,
    "msg":"新增工单失败"
}
```

- 请求方法：delete
- url: `/productionOrder/delete/{id}`
- 参数： 请求参数在请求路径中，{id}为工单的id
- 响应结果：

正确

```json
{
    "code":200,
    "msg":"删除工单成功"
}
```

失败：

```json
{
    "code":500,
    "msg":"删除工单失败"
}
```
